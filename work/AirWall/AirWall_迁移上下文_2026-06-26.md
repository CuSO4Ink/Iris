## 1. 项目目标

项目是一个 Unreal Engine 的 **空气墙 / 能量墙（AirWall）** 系统。

核心目标：

1. 使用 spline 生成空气墙。
2. 碰撞层用自适应分段 Box Collision 贴合曲线。
3. 视觉层用 `SplineMesh` 表现。
4. 支持多层视觉 mesh，方便不同特效层叠加。
5. 使用 VFX Uber 母材质 + MI 配置不同表现。
6. 支持玩家撞击交互：
   - 方案 A：把交互信息传给材质，让材质内部响应。
   - 方案 B：在撞击位置直接 Spawn Niagara / 独立特效。
7. 希望蓝图只提供通用交互 payload，具体视觉表现交给特效 MI / 材质函数配置，避免随意修改母材质。

## 2. 当前已完成状态

### 2.1 碰撞分段

- 已实现并跑通自适应 step 分段。
- v1 是固定 `MaxStep`；后续优化为基于控制点段长 / 两点间线段比例二分。
- 用户后来又手动加回一个步进最大值，用于防止 S 弯中点采样漏判。
- 碰撞 Box 当前能正常生成，且已跑通。

### 2.2 视觉 SplineMesh

- 已有 `GenerateSplineMesh` 类似函数。
- 函数输入里的 `StartP` / `EndP` 是 float，含义是沿 spline 的距离值，不是 Vector。
- 函数内部会用类似：
  - `GetLocationAtDistanceAlongSpline(StartP)` 得到 Vector 位置。
  - `GetTangentAtDistanceAlongSpline(StartP)` 得到 tangent。
  - `GetRightVectorAtDistanceAlongSpline(StartP)` 可得到侧向方向。

### 2.3 多层视觉 mesh

- 需求：碰撞只保留一层，视觉 `SplineMesh` 可以叠多层。
- 多层用于：底部火苗层、主墙层、高光/折射层等。
- 已讨论：
  - `VisualLayerCount`
  - `LayerSpacing`
  - `LayerMaterials : Material Interface Array`
  - `VisualLayerMIDs : MaterialInstanceDynamic Array`
- 注意：`LayerMaterials` 可以只放一个默认 MI，所有层 fallback 到 `[0]`。
- 每层应创建独立 DMI，但不要每段创建 DMI。

正确思路：

```text
每层一个 DMI
所有该层的 spline mesh 段共享这个 DMI
```

不要做成：

```text
每段 × 每层 一个 DMI
```

否则 20 段 × 3 层 = 60 个 DMI，交互参数要写 60 次。

## 3. 自适应碰撞分段上下文

### 3.1 弦高 / sag 判据

一段曲线用一个直 Box 近似。`sag` 表示曲线和直弦之间的最大偏差，单位 cm。

早期简化计算：

```text
P0 = 段起点
P1 = 段终点
Cmid = (P0 + P1) / 2
Smid = GetLocationAtDistanceAlongSpline((D + D1) / 2)
sag = VSize(Smid - Cmid)
```

`sag <= Tolerance` 则接受该段，否则 step 砍半继续细分。

### 3.2 S 弯采样陷阱

如果一段是 S 弯，曲线中点可能刚好穿过弦，导致：

```text
Smid ≈ Cmid
sag ≈ 0
```

这样会误判为直段。

用户当前处理方式：

- 按两点之间线段比例进行二分。
- 手动设置一个 `MaxStep` / 步进最大值，强制长段先切开。
- 这样可以把完整 S 弯拆成两个单弯，避免中点采样被对称性骗过。

补充建议：如果以后要进一步稳，可以改为三点采样取最大：

```text
t = 0.25 / 0.5 / 0.75
sag = max(sag25, sag50, sag75)
```

但当前用户已能接受 `MaxStep` 在精度和格子数量之间手动平衡。

## 4. 材质 / UV 系统上下文

### 4.1 当前材质函数

用户已有材质函数，类似 `MF_AdvancedSplineUV`。

当前蓝图通过 `SetDefaultCustomPrimitiveDataVector2(DataIndex=0)` 给每段 `SplineMesh` 写入 CPD：

```text
CPD0 = UStart
CPD1 = UEnd
```

旧版本曾经是：

```text
CPD0 = StartP / Divisor
CPD1 = EndP / Divisor - 1
材质里再 EndOffset + 1
```

建议已提出：去掉 `-1/+1` 魔法数，直接存真实 `UEnd`：

```text
CPD0 = UStart
CPD1 = UEnd
U_Global = lerp(UStart, UEnd, BaseUV.R)
```

### 4.2 CPD 与 MID 分工

CPD 适合每段不同的数据：

```text
CPD0 = UStart
CPD1 = UEnd
```

MID 适合每层 / 整面墙共享的数据：

```text
WallBaseZ
VClampOffset
VClampHeight
VTileWorldSize
InteractionStartTime
InteractionStrength
InteractionRadius
```

原因：同一层所有段共享一个 MID；如果用 MID 存 `UStart/UEnd`，后一个段会覆盖前一个段。

### 4.3 V 轴采样模式

用户不想所有效果都纵向重复，因此决定只输出一个最终 UV，但 V 由 StaticSwitch 选择：

```text
U_Global = lerp(CPD0, CPD1, BaseUV.R)

HeightFromBottom = AbsoluteWorldPosition.Z - WallBaseZ

VClampOnce = saturate((HeightFromBottom - VClampOffset) / VClampHeight)
VRepeatWorld = HeightFromBottom / VTileWorldSize

V_Final = StaticSwitch(UseClampV ? VClampOnce : VRepeatWorld)

UV_Final = float2(U_Global, V_Final)
```

解释：

- `VClampOnce`：固定竖直范围内采一次，适合底部火苗、渐变、hero mask。
- `VRepeatWorld`：按世界高度重复，适合噪声、流光、网格、扫描线。
- `BaseUV.G` 在这个方案里不再用于 V，V 完全由世界高度算。

### 4.4 WallBaseZ

若空气墙 Actor 拖到地面，且墙从 Actor 位置往上长：

```text
WallBaseZ = GetActorLocation.Z
```

如果 Actor 不在地面，则用 spline 第一个点世界 Z：

```text
Spline.GetLocationAtSplinePoint(0, World).Z
```

## 5. 多层视觉 mesh 设计

### 5.1 核心原则

```text
Collision：单层
Visual：多层
```

碰撞只负责阻挡和 Hit，不要复制多层。

### 5.2 多层偏移

之前有一次误解：我曾说 `StartP + LayerOffset`，但用户指出 `StartP` 是 float spline distance，不是 Vector。

正确区分：

- 如果 `LayerOffset` 是 float，加到 `StartP` 上，效果是沿曲线方向前后滑动，不是真正侧向偏移。
- 真正侧向偏移应该加在 `GetLocationAtDistanceAlongSpline` 返回的 Vector 位置上。

推荐做法：

```text
调用 GetFullSplineData(StartP, EndP)
得到 SP / EP（Vector）
Right = GetRightVectorAtDistanceAlongSpline(StartP)
LayerOffset = Right * LayerOffsetAmount
SP_Layer = SP + LayerOffset
EP_Layer = EP + LayerOffset
Tangent 不加 Offset
```

不过如果墙基本直、Offset 很小，`StartP + floatOffset` 看起来也可能“对”，但本质是沿曲线滑动，不是横向叠层。

### 5.3 DMI 创建策略

不要在 `GenerateSplineMesh` 内部每次创建 DMI。

推荐顺序：

```text
Construction / 生成开始：
  Clear VisualLayerMIDs
  For LayerIndex = 0 -> VisualLayerCount - 1:
    选择 Source Material
    CreateDynamicMaterialInstance
    Set 静态参数（WallBaseZ / UV 参数等）
    Add 到 VisualLayerMIDs

生成每段每层 mesh 时：
  Get VisualLayerMIDs[LayerIndex]
  SetMaterial 给当前 SplineMesh
```

`LayerMaterials` 选择逻辑：

```text
if LayerMaterials 有 LayerIndex:
    SourceMaterial = LayerMaterials[LayerIndex]
else if LayerMaterials 有 0:
    SourceMaterial = LayerMaterials[0]
else:
    SourceMaterial = DefaultLayerMaterial
```

## 6. 玩家交互系统上下文

### 6.1 当前蓝图状态

用户最新 `s.txt` 解析结果：

```text
OnWallHit
 → Branch 节流
 → ForEach VisualLayerMIDs
      → SetVectorParameterValue("ContactWS", ImpactPoint)
      → SetScalarParameterValue("RippleStart", GetGameTimeInSeconds)
 → Completed 后又写 FallbackMID
 → Set LastRippleTime
```

当前链路作为材质响应入口是对的，但命名仍是波纹 demo 语义。

### 6.2 建议泛化为 InteractionPayload

不要继续扩展 `Ripple` 命名。建议新增：

```text
InteractionPositionWS
InteractionStartTime
InteractionSurfaceU
InteractionHeight
InteractionStrength
InteractionRadius
InteractionNormalWS（可选）
```

蓝图只负责提供 payload，不负责具体特效表现。

### 6.3 解决弧形墙对面扩散 bug

曾发现 bug：C/O 形墙撞一处，对面也扩散。

真因：材质用 3D 距离：

```text
Distance(AbsoluteWorldPosition, ContactWS)
```

这会穿透墙体几何。弧形墙对面虽然沿墙很远，但空间直线距离近。

根治：改用墙表面 2D 距离：

```text
dist = distance(
  float2(PixelSurfaceU, PixelHeight),
  float2(InteractionSurfaceU, InteractionHeight)
)
```

蓝图需要在 Hit 时算：

```text
InteractionSurfaceU = FindDistanceClosestToWorldLocation(ImpactPoint)
InteractionHeight = ImpactPoint.Z - WallBaseZ
```

UE 蓝图节点可能是组合：

```text
Find Input Key Closest To World Location
→ Get Distance Along Spline At Spline Input Key
```

### 6.4 两套交互表现方案

#### 方案 A：材质内响应

蓝图：

```text
ForEach VisualLayerMIDs:
  Set InteractionXXX 参数
```

材质：

```text
MF_VFX_InteractionSlot
```

输出通用 mask：

```text
InteractionAge
InteractionDist
InteractionFade
InteractionMask
InteractionRingMask
```

特效 MI 自己决定这些 mask 接到哪里：

```text
Emissive
Opacity
Refraction
NoiseStrength
ColorLerp
DissolveThreshold
```

#### 方案 B：Spawn 独立特效

蓝图在交互点直接：

```text
Spawn Niagara System At Location
Location = ImpactPoint
Rotation = ImpactNormal 对齐
User 参数 = InteractionStrength / Radius / Direction 等
```

适合火花、爆点、一次性特效。

### 6.5 FallbackMID 建议

当前图里 ForEach `VisualLayerMIDs` 后又写 `FallbackMID`。

正式多层后建议改成：

```text
if VisualLayerMIDs.Length > 0:
    只写 VisualLayerMIDs
else:
    写 FallbackMID
```

否则多层存在时还写旧 `FallbackMID`，调试会混乱。

## 7. 交互 contract 边界

用户希望：只提供位置和时间信息，剩下交给特效自由发挥，但不能随意改母材质。

建议边界：

### 蓝图维护

```text
InteractionPayload 参数名
OnWallHit 节流
SurfaceU / Height 计算
写入每层 MID
可选 Spawn VFX
```

### 母材质维护

```text
MF_VFX_InteractionSlot
固定参数名
固定输出 mask
少量 StaticSwitch
```

### 特效可自由配置

在 MI 内调：

```text
颜色
强度
半径
宽度
速度
持续时间
贴图
噪声强度
Opacity/Emissive/Distortion 权重
LayerMaterials 每层挂哪个 MI
```

### 不允许随意改

```text
母材质拓扑
CPD 分配
Interaction 参数名
过多 StaticSwitch
多事件数组
```

原则：自由发生在 MI 参数和预留插槽里，不发生在 Uber 母材质主干拓扑里。

## 8. 当前关键坑位总结

1. `Break Hit Result` 里的 `Time` 不是游戏时间，不要用。游戏时间用 `Get Game Time in Seconds`。
2. `OnComponentHit` 贴墙会每帧狂触发，必须节流。
3. 多层视觉 mesh 不要复制碰撞。
4. DMI 应该每层一个，不是每段一个。
5. CPD 适合每段不同值，MID 适合每层共享值。
6. `ContactWS` 3D 距离会导致弧形墙对面响应，材质响应应改用表面 2D 距离。
7. `FallbackMID` 多层后只应作为 fallback，不应和数组同时写。
8. 多层半透明会增加 overdraw，LayerCount 默认建议 2-3，后续必须看 `Shader Complexity` 和 `Quad Overdraw`。
9. 贴图 clamp 边缘灰缝可能来自 mipmap / 边缘像素，mask 贴图可尝试 `NoMipmaps` 或保证边缘纯黑。
