# research — 专题调研

**有明确课题范围**的深入调研，产出一份相对完整的报告 / 对比 / 结论。

## 放什么

- 技术选型对比（A vs B vs C，带结论）
- 论文 / 资料的系统性解读
- 某个方向的探索性研究（"能不能用 X 解决 Y"）
- 可复用的调研结论，供未来项目引用

## 不放什么

- 零散的知识点 → `notes/`
- 绑定具体项目的方案设计 → `work/<项目名>/`（项目内部也可以做调研，但成果就留在该项目目录里）

## 组织建议

- 每个课题一个子目录或一个文件，文件名点明主题
- 调研结果建议在开头写一段 **TL;DR / 结论**，方便将来快速回忆
