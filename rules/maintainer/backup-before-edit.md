# Backup Before Edit — 改前备份准则

> **适用**：维护者 AI 改动 `rules/` / `templates/` / 根 `README.md` / 各目录 `README.md` / `USAGE.ankoha.md` 等**结构性文件**之前。
> **不适用**：新建文件、项目 AI 在 `work/<项目>/` 内的普通操作、memory 文件追加。

## 规则

动手改任一结构性文件前，先把它复制一份到：

```
.workbuddy/backups/<YYYY-MM-DD>/<原文件名>.bak
```

- 日期目录不存在就创建
- 同一天多次改同一文件：保留**最早**那份 `.bak`（代表当天初始状态），不覆盖
- 备份做完再改原文件

## 为什么

- 结构性文件是规则/模板/入口，改坏了影响面大
- 备份成本低（一条 `cp`），回滚价值高
- `.workbuddy/` 不在 vault 主视野中，备份不污染 PARA 结构

## 什么情况可以跳过

- 只加新章节不删旧内容，且改动 ≤ 3 行
- 纯 typo / 标点修复
- 新建文件（没有旧版可备）

跳过时要在回复里明说一句"本次改动小，跳过备份"，让用户知道是**主动判断**而非遗忘。

## 清理

- `.workbuddy/backups/` 下超过 30 天的目录可删
- 需要长期保存的关键版本，移到 `archive/rules-history/` 或自行处理
