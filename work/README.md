# work — 活跃项目工作区

每个**进行中的项目**一个子目录，结项后整个目录搬到 `archive/<项目名>/`。

## 组织约定

- 一个项目一个子目录，目录名用项目代号（`DyeSplashBaker/`、`FloodStep/` 等）
- 项目内部文档命名建议：
  - `SWE-TechSpec.md` / `<项目>-方案.md` — 技术方案主文档
  - `README.md` — 项目简介、当前状态（可选，但多文档时建议加）
  - `assets/` — 项目专属图片 / 附件（不共用就放这里，不要污染根 `assets/`）
- 项目内调研也留在项目目录里，不要挪到 `research/`（除非结论通用）

## 与其他目录的边界

- **不是**项目相关的长期笔记 → `notes/`
- 通用调研结论 → `research/`
- 未消化的碎片 → `inbox/`
- 项目结项 → 整个目录 `mv` 到 `archive/`
