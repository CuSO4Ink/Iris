# UE Neural Render Lab

> **L2 项目身份**。接手本项目的 AI 必读。

## 一句话介绍

探索「UE 小模型部署 / UE 内置神经渲染能力 / 神经渲染后处理管线」接入现有 UE 生成管线的可行性，目标是分清哪些能实时落地、哪些只适合离线生成、哪些目前不值得做。

## 当前状态

活跃 · 项目初始化

## 当前焦点

先做技术侦察与可行性矩阵：把候选路线按接入点、推理方式、性能预算、资产生产成本、与现有生成管线耦合程度排优先级。

## 技术栈与硬约束

- 默认目标 UE 版本：UE5.8 源码版；Windows + Rider 环境。
- 运行时优先级：实时编辑器预览 > Sequencer / Movie Render Pipeline 离线渲染 > 纯研究 demo。
- 小模型优先考虑 UE 内置或官方方向：NNE / ONNX / RDG Compute / SceneViewExtension / Post Process 接入；避免一开始绑死重型 Python 服务。
- 首轮模型约束：优先验证 <= 10MB、FP16/INT8 友好、单帧预算可控的小网络；不追求大模型端到端生成。
- 评价指标必须量化：1080p 与 4K 下 GPU ms、显存、输入输出纹理尺寸、延迟帧数、资产制作/训练成本。

## 候选路线

| 路线 | 假设 | 初始判断 |
|---|---|---|
| UE 小模型部署 | 用 NNE / ONNX Runtime 类路径跑轻量 CNN/MLP，输入 SceneColor/Depth/Normal/GBuffer 派生特征 | 最值得先验证，边界清晰，能给出 ms 预算 |
| 神经后处理材质 | 用 Post Process Material + 纹理/LUT/小模型近似神经风格化、去噪、超分或材质重映射 | 材质本身不是神经网络，但适合做「神经结果的轻量运行时表达」 |
| RDG/Compute 神经 pass | 自定义渲染 pass 读写 RenderTarget，跑 tiled inference 或 compute shader | 潜力最大，但工程成本高，第二阶段再碰 |
| 离线神经渲染管线 | Python/外部推理生成贴图、体积、点云、风格化序列，再导入 UE | 生产可行性高，但不是实时 UE 内部署；适合做对照组 |
| UE 自带神经渲染相关能力 | 盘点 NNE、ML Deformer、Neural Network 推理、官方插件/实验特性 | 先做事实核查，不凭记忆下结论 |

## 术语表

- **小模型**：为 UE 运行时准备的轻量神经网络，通常关注模型体积、推理延迟、输入输出纹理格式。
- **神经后处理**：在渲染后处理阶段使用神经模型或神经模型蒸馏结果改变画面。
- **生成管线**：当前 AI/程序化效果从规格、素材、Niagara/材质/蓝图到 UE 渲染输出的生产链路。
- **接入点**：模型或神经结果进入 UE 的位置，例如 Post Process Material、SceneViewExtension、RDG pass、Movie Render Pipeline、资产导入阶段。
