## §A 顶视图要画的 7 样东西（照着填）

一张俯视图（纸上手绘 / Figma / PS 均可），从玩家出生端（S1，0m）到回望端（S4，200m）单向铺开。必须标出：

1. **路径中心线** —— 玩家实际走的那条线（可弯，不必直），从 0m 画到 200m。
2. **4 段边界** —— 在 50m / 120m / 160m 三处画横切线，切出 S1/S2/S3/S4。
3. **两侧不可越边界** —— 左右各画一条：哪侧是悬崖 mesh、哪侧是海面 FluidFlux（或两侧都悬崖，海在正前方）。标注沙滩宽度（20-40m 之间，可变宽变窄制造节奏）。
4. **4 个视觉锚点位置** —— 每段放 1 个主锚点（吸引视线的巨构/遗迹/恒星），标在图上打 ★。
5. **每个锚点的机位朝向** —— 从锚点画一个箭头，表示"玩家走到这里时镜头大致朝哪看"。S3 那个就是主机位⑤方向。
6. **玩家停留秒数** —— 每段标一个数字：玩家大约在此停留几秒（全程加起来 30-60s）。
7. **恒星在图上的方位** —— 恒星不在地面，但要标它投影在地平线的方位角（相对路径），确保 S1 建立、S3 hero、S4 回望都能看到它（走位验证的依据）。

## §D 验收清单（画完逐条自查，全过才进 T-P0.1）

> 注：T-P0.0 只验"图纸层面的规划完整性"。带 ⏸ 的量化项（FOV/屏占比等）留到 T-P0.1 灰模阶段用占位球/小白人实测，此处只需在图上有**规划意图**即可。

**图纸完整性（现在必过）**
- [ ] 顶视图含 §A 全部 7 要素（中心线/4段边界/两侧边界/4锚点/机位朝向/停留秒数/恒星方位）
- [ ] 4 段边界切分明确，S1/S2/S3/S4 位置与长度定死
- [ ] 每段有且仅有 1 个主锚点，位置在图上标注
- [ ] 4 张 thumbnail 完成，每张视觉重心 + 恒星位置 + 前进钩子清晰
- [ ] §C 关键参数尽量填满（至少段边界 X + 各段宽度 + 恒星方位角 + 主机位站位）

**叙事弧线自洽（现在必过，对照 CONCEPT §4）**
- [ ] S1 出生即能看到恒星（建立镜头成立），有明确"往前走"的引导
- [ ] S3 是叙事高潮：恒星 hero + 主机位⑤最优构图点，停留秒数最长
- [ ] S4 回望时能重新看到远景/恒星（收束成立）
- [ ] 全程停留秒数加总落在 30-60s

**跑图硬约束（现在必过）**
- [ ] 路径全程单向、可通行，无预期的掉海/穿墙/死胡同
- [ ] 两侧边界玩家不可越（悬崖或海）
- [ ] 恒星在 S1→S4 移动全程都在视野内（走位验证的规划依据，非只测 S3）

**留到 T-P0.1 灰模实测（现在只需图上有意图，标 ⏸）**
- [ ] ⏸ 主机位⑤ FOV 24-35mm 等效
- [ ] ⏸ 恒星屏占比 30-45%（W2 前用占位发光球复验）
- [ ] ⏸ 人物剪影屏占比 3-8%（用默认小白人量）
- [ ] ⏸ 景深三档分层（前/中/远景占位）
- [ ] ⏸ 背景光比（恒星 vs 中景遗迹 EV 差，W2 恒星实体后量）
