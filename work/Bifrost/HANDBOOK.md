## §0 三方协作总则（先读这段，全手册都在用）

本项目有**三个角色**，全程用固定符号标注每步归谁：

| 符号 | 角色 | 干什么 | 不干什么 |
|---|---|---|---|
| 🤖 | **对话 AI（我）** | 出 Python 脚本、MCP 指令、材质参数建议、提示词、报错诊断、决策分析 | **不直接碰 UE**（我看不到你的视口，只能给"可执行物"） |
| 🎮 | **UE-Agent**（连 MCP 那个 Agent） | 在 UE 里实际执行：CreateMaterial / PlaceActor / `script.execute`（经 `UE_MCP_Access_Pack` 网关，脚本内用 `execute_tool` 不 `import unreal`）/ TakeScreenshot | 不做审美判断（它只执行，不替你选好看） |
| 🧑 | **你（TA）** | 拍板审美 · blockout 摆放 · 材质节点最终精修 · LookDev 调色 · Sequencer 录像 · **所有验收** | 不写你不熟的底层 C++（也不需要） |

### 核心循环（记住这一张图，全项目都在转它）

```
🧑 你提需求/给锚点
      ↓
🤖 我出脚本 or MCP 指令（可复制粘贴）
      ↓
🎮 UE-Agent 在 UE 里执行
      ↓
🧑 你验收（看视口 / 让 Agent 截图）
      ↓  ┌─ 过 → 下一步
      └─ 不过 → 把现象/报错回传给 🤖 → 我改脚本 → 🎮 重跑
```

> **关键心法**：你不该"自己写脚本硬啃"，也不该"让 Agent 瞎试"。正确姿势是——**你只管提需求和验收，脚本永远找我要，执行永远交 Agent**。你的时间只花在 AI 替代不了的地方（见 §8 红线清单）。

### MCP 铁律：每次让 🎮 批量生成，必带这 4 件（可回滚可清理）

任何批量 spawn / 批量建材质，下达时**必须**包含：

1. **batch tag**：本批唯一标识，如 `batch=W2_kitbash_S3_v1`
2. **semantic tags**：语义标签，如 `tags=[rock, midground, S3]`
3. **Outliner folder**：归到指定文件夹，如 `folder=/Bifrost/S3/Kitbash`
4. **cleanup 方案**：配套一段"按 batch tag 删除本批"的脚本，跑砸了能一键回滚

**给 🎮 的下达模板（复制改参即可）：**

```
请执行以下批量操作，严格遵守 MCP 铁律：
- batch tag: <本批唯一名>
- semantic tags: <语义标签列表>
- Outliner folder: <目标文件夹路径>
- 操作内容: <spawn 什么 / 建什么材质 / 参数范围>
执行完请 TakeScreenshot 回传，并告诉我：本批实际生成了几个、是否有报错。
另外给我一段“按此 batch tag 清理本批”的 cleanup 脚本备用。
```
