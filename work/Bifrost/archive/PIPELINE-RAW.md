## 一、等离子恒星（A1，hero + 光源）怎么做

**分三层堆，从便宜到贵：**
1. **本体表面**：大球 mesh + 自定义材质。翻涌熔融感用 **多层 3D 噪声（domain warp 扭曲）+ 时间演化** 做流动，配 emissive 拉出冷白过曝核心 + icy cyan 边缘。**这是主力，纯材质，最省。**
2. **isosurface 细节（可选加强）**：复用你的 NS_Slime marching cubes，只在**边缘轮廓/日珥弧**处叠真 dynamic mesh，做出"融合翻涌"的立体感——不整球都上，成本可控（呼应 TECH-SCOUT A方案"轮廓局部化"）。
3. **作为光源**：恒星驱动一个方向光/大范围光 + 大气散射，让它**真的照亮整个海岸**——这是"恒星驱动一切"因果链的物理落点。

**关键控制**：D2 冷神性红线——核心可过曝但要留住 pale azure→icy cyan 的色彩梯度，别糊成纯白。

## 二、天象（A2极光 + A3体积云）怎么做

**统一走一套 raymarching 体积框架**（TECH-SCOUT 的"管线A—天空是一个体积场"思路），一套框架喂不同介质，省重复造轮子：

| 天象 | 做法 | 成本 |
|---|---|---|
| **A3 体积云/贝母云** | raymarch + 3D 噪声密度场，恒星光穿透散射。**最稳的底，先做** | 中 |
| **A2 极光** | 同框架，改成沿"磁力线"分布的发射介质 + 冷绿/冷紫，做**恒星磁场的可视化衍生**（又一条因果链） | 中高 |
| **god-rays/大气散射** | 恒星光 + 体积雾自然产出 | 低（恒星衍生） |

**因果链收束**：恒星光→云的散射、恒星磁场→极光、恒星磁场→海面尖刺——**天象和海面全是恒星的衍生**，叙事高度自洽。

## 三、场景布置（巨构悬崖遗迹）怎么做

**主机位已锁⑤前景剪影环境镜**，所以布置**围着这个机位优化**，不做全关卡：
1. **构图骨架先行**：先用 blockout（灰模大块）摆悬崖/遗迹/人物剪影/恒星的**位置和比例关系**，确认"渺小人 vs 压迫性巨构"的构图成立——**这步最重要，先灰模定构图再做资产**。
2. **kitbash 巨构**：岩石母材质 + 模块化遗迹件拼装，L1 量产走 MCP。
3. **景深分层**：前景（人物剪影+近岩）/ 中景（巨构遗迹）/ 远景（恒星+天象）——三层各自的资产精度不同，远景可低模。

# 美术生产 & 执行管线（可落地版）

这套管线的**灵魂是 L1/L2/L3 三层分工 + MCP 量产**，也是 Bifrost 作为 P3 代表作要展示的"AI提效工作流"本身。

## 阶段管线（按依赖排序）

```
P0 构图blockout（人，最先）
   └─ 灰模摆巨构/人物/恒星比例，锁主机位构图
        ↓
P1 L1量产层（MCP/Agent 走，人验收）
   ├─ 岩石母材质+MI变体   ┐
   ├─ 模块化遗迹kitbash件  ├─ MCP execute_python_script 批量生成/摆放
   ├─ 植被/天空球/远景mesh ┘
   └─ FluidFlux 主海面接入（已复刻，直接用）
        ↓
P2 L3内核层（人，做深）
   ├─ 等离子恒星（材质+可选isosurface+光源）
   └─ 天象体积框架（云→极光）
        ↓
P3 点缀模块（独立挂载，有余力才做）
   ├─ 铁磁流体尖刺（材质①+局部Niagara②，挂FluidFlux）
   ├─ 分色/沙雪/鸟群…
        ↓
P4 L2审美层（人，整合，不可压缩）
   └─ LookDev + 冷神性调色 + 后期 + 最终构图微调
        ↓
P5 breakdown素材整理（全程留，末尾汇总）
```

## MCP 量产管线（P1 的核心，也是要展示的工作流）

复用你 UEAgent 的 `execute_python_script` / `execute_unreal_command` 链路：
- **母材质生成**：定义参数面板规格（冷神性色域）→ 脚本批量建材质 + MI 变体 → 人 LookDev 验收
- **资产摆放**：脚本批量 spawn/transform 远景 mesh、植被散布 → 人调构图
- **规范约束**（你记忆里的 UE MCP 铁律）：每批生成必设 **batch tag + semantic tags + Outliner folder + cleanup 方案**，保证可回滚、可清理

## 人 vs AI 边界（Human-in-the-loop）
| 层 | 谁 | 做什么 |
|---|---|---|
| L1 量产 | **AI/MCP** | 母材质、kitbash 件、植被、远景摆放、参数脚手架 |
| L2 审美 | **人** | 构图blockout、LookDev、调色、比例、最终把控 |
| L3 内核 | **人** | 恒星、天象、铁磁流体融合——审美强相关，人手调 |

## 我的判断
这套管线的关键取舍：
1. **构图 blockout 先行**——先灰模锁构图再做资产，避免做完资产发现构图不成立的返工。
2. **一套 raymarching 框架喂多天象**——不为极光/云各造一套。
3. **MCP 量产本身就是要展示的产物**——P1 的脚本和规范不是幕后工具，是 P3 breakdown 的核心素材。
4. **点缀全独立挂载**——P3 每个模块可插可拔，不拖累 P0-P2 基底交付。
