# Project Template Kit — 项目工作台模板组

> **用法**：新建项目时用 `/project-init <项目名>` 命令，或手动把本目录下三份模板复制到 `work/<项目名>/`。
> **定位**：L2 项目专属，叠加在 L0 之上。

## 三件套

| 文件 | 定位 | 更新节奏 | 体积 |
|---|---|---|---|
| `AI-BRIEF.md` | 项目身份、技术栈、术语、当前焦点 | 阶段性（周/月） | ≤100 行 |
| `LOG.md` | 决策流水与否决方案（追加式） | 高频（每次 `/log`） | 追加无上限 |
| `BACKLOG.md` | 待办清单（简单打勾） | 随时 | 一行一项 |

## 填充原则

- **AI-BRIEF**：锚点类信息，变动频率低，AI 每次接手必读
- **LOG**：只记"本来会忘"的东西——决策、否决方案、意外发现。不记流水账
- **BACKLOG**：顺手加、做完打勾、完成的可归入 LOG
- **不要新增第四个文件**，除非某个具体任务大到需要 `tasks/T-xxx.md`（按需创建）
