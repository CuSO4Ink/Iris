# archive — 归档区

已结项 / 已废弃的项目、规则、文档。**保留但不再维护**，供将来查阅或复用局部思路。

## 组织约定

- **与 `work/` 对称**：每个归档项目一个子目录，直接从 `work/<项目名>/` 整体 `mv` 过来
- 归档规则放 `archive/rules/<topic>.md`
- 归档文档在开头建议加一段"归档说明"，包含：
  - **归档日期**
  - **归档原因**（废弃了？被替代了？项目结束了？）
  - **后续方案**（如果被替代，指向新方案的路径）

## 不放什么

- 还在维护 / 可能重启的项目 → 留在 `work/`
- 单纯过时的知识笔记 → 直接在 `notes/` 里删或原地标注"过时"，不用搬

## 取回

从 `archive/` 把目录 `mv` 回 `work/`，并在项目主文档里删掉"归档说明"段落。
