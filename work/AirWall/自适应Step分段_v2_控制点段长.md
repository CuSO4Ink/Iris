# 空气墙碰撞分段 v2：按控制点段长自适应

> 优化方案 · AirWall 项目 · 2026-06-05
> 承接 v1（弦高自适应步长），核心改动：`MaxStep` 从写死常量 → "当前控制点段长"

## 一、v1 的残留浪费

v1 用弦高判据 + 二分细化已经解决了"直段浪费、弯段穿漏、末端外推"三大硬伤。但它有一个**人为拍脑袋的参数没拔掉**——`MaxStep`（直段单步上限）。

它带来一个隐藏浪费：

```
碰到一段很长的直墙（比如两个控制点之间拉了 5000cm 的直线）
→ 固定 MaxStep=600 逼着它至少切成 5000/600 ≈ 9 个 Box
→ 可这段是笔直的，本该用【一个】Box 带过
```

本质问题：**直段的单步上限不该是个固定数，而应该等于"这段直墙自然有多长"**。这个"自然长度"，spline 早就替我们标好了——就是相邻控制点之间的距离。

## 二、核心思路：让控制点段长当 MaxStep

把外层从"沿 spline 固定步长前进"换成"**遍历相邻控制点对**"。每对控制点 `(i, i+1)` 之间是一段自然的曲线，它的长度就是这一段的天然步长上限：

- **直段**：第一刀用整段长度试探 → 弦高直接达标 → 一个 Box 带过，二分根本不进
- **弯段**：第一刀弦高超标 → 进二分砍半，和 v1 一样细化贴合

`MaxStep` 不再是常量，而是**每段动态取的"剩余段长"**。直段有多长就一步多长，弯段该多密还多密。

## 三、算法结构：两层 → 三层

v1 是两层（外层前进 + 内层二分）。v2 在最外面套一层"控制点遍历"：

### 第①层 For Loop —— 遍历控制点对
```
i 从 0 到 GetNumberOfSplinePoints − 2
  D_start = GetDistanceAlongSplineAtSplinePoint(i)
  D_end   = GetDistanceAlongSplineAtSplinePoint(i+1)   ← 这一段的自然边界
```
> 闭合 spline：上限取 `N−1` 并对最后一段（末点→首点）单独处理。

### 第②层 While —— 在当前段内前进
```
条件：(D < D_end) AND (guard < 4096)
段内沿用 v1 的前进逻辑：每段结束 D = D1
```
> `guard` 保命栓保留，防任何意外恒真。

### 第③层 While —— 二分细化（和 v1 一样，只改 step 初值）
```
step 初值 = min( D_end − D , MaxStepSafety )   ★关键改动
进环前用这个 step 先算一次 sag
条件：(sag > Tolerance) AND (step > MinStep)
体内：step *= 0.5，重算 sag
退出：sag 达标 或 step 触底 MinStep
```

直段：`step 初值 = 剩余整段长`，第一次 sag 就 ≤ Tolerance → 不进循环体 → 一个 Box。

### EMIT —— 不变
```
Box 位置 = 段中点
Box 朝向 = FindLookAtRotation(起点 → 终点)  （高低差场景按需只取 Yaw）
Box 尺寸 = SetBoxExtent(段长/2, 墙厚/2, 墙高/2)   ← Scale 仍必须 = 1
```

## 四、必须保留的兜底：MaxStepSafety

这是新方案唯一容易漏的点，**不能省**。

**为什么需要**：弦高用的是"中点单点采样"。一段三次曲线（Hermite/Bézier）**最多能拉出 S 形（两个弯）**。如果某两个控制点离得特别远（几千 cm）、中间又被切线拉成 S 形，那么中点恰好落在曲线穿过弦的位置 → `sag ≈ 0` → **误判为直段 → 漏切**。

v1 的固定 `MaxStep` 其实**无意中当了安全网**（强制每步不超过几百 cm，一段内不会出现 S）。v2 第一刀直接用整段长，把这张网撤了，所以要手动补回来：

```
step 初值 = min( D_end − D , MaxStepSafety )
MaxStepSafety = 2000（给个很大的值）
```

平时根本碰不到这个上限（正常控制点段就几百 cm）；只有遇到"几千 cm 还绕 S 形"的病态段，它强制先切一刀 → 逼出第二次、第三次采样 → S 被切成两段单弯 → 采样恢复可靠。**两全其美：正常直段一步到底，病态长段不漏缝。**

## 五、效果对比（v1 → v2）

| | v1 固定 MaxStep | v2 控制点段长 |
|---|---|---|
| 长直段 | 强制切 `长度/MaxStep` 个 Box | **一个 Box 带过** |
| 弯段精度 | 弦高二分贴合 | **不变，照样贴合** |
| Box 边界 | 落在固定步长刻度上 | **落在控制点上**（曲率突变处） |
| 直段参数 | 要调 MaxStep 平衡疏密 | **不用调，段长自动当上限** |
| S 形隐患 | MaxStep 当安全网 | 靠 MaxStepSafety 兜底 |

## 六、附带白赚的好处

Box 边界天然落在**控制点**上，而控制点正是**曲率最可能突变**的地方。在那里强制断一刀，相邻 Box 各自只覆盖一段曲率连续的曲线：

- 朝向（`FindLookAtRotation` 高低差接缝问题）更服帖
- 接缝错位概率更低
- 这个优化顺手把斜坡接缝隐患也缓解了一点

## 七、新增节点清单

| 节点 | 作用 |
|---|---|
| `GetNumberOfSplinePoints` | 拿控制点总数（For Loop 上限） |
| `GetDistanceAlongSplineAtSplinePoint(i)` | 把"第几个控制点"翻译成"沿曲线里程"，接已有的 `GetLocationAtDistanceAlongSpline` |

## 八、参数（v2）

| 参数 | 经验值 | 含义 | 相对 v1 |
|---|---|---|---|
| `MaxStepSafety` | 2000 cm | 病态长段兜底上限 | **替代** v1 的 MaxStep |
| `MinStep` | 20~50 cm | 弯段最小步长 | 不变 |
| `Tolerance` | 5~15 cm | 唯一画质旋钮 | 不变 |

一句话：**v2 干掉了 v1 里"调 MaxStep 平衡疏密"这件事，直段自动一步到底，弯段精度不变，代价只是多一层控制点遍历 + 留一个大兜底。**
