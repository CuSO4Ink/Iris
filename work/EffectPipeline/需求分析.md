# EffectPipeline · 需求分析与导入规范

> 立项需求分析。创建 2026-06-29，2026-06-30 对齐实勘结论精简。
> 实勘细节见 `notes/外包归档清单.md`、`notes/ZibraVDB排查结论.md`。

## 1. 需求

外包（漫行者）交付 20+ 个 Houdini+UE 特效，形态不统一。要做两件事：
1. **规范导入**：以统一规范并入**已有主工程**（自研 UE 5.7.3，不新建）。
2. **统一材质**：建母材质 + MI 替换外包散材质。

约束：保留 Houdini Engine（含 HoudiniNiagara）+ ZibraVDB 插件。

## 2. 现状痛点（实勘修正）

| 维度 | 实勘结论 |
|---|---|
| 交付形态 | 15 个 uproject，完整度分级：A 级完整 / B 级残缺（见归档清单） |
| 引擎版本 | **统一 UE 5.5 公版**（C30=GUID自定义）；主工程是自研 5.7.3 → 跨版本+跨引擎 |
| 插件依赖 | HoudiniNiagara(引擎自带) / HoudiniEngine(工程级) / ZibraVDB(预编译二进制) |
| 残缺包 | D20 等只给散资产，缺 Config + ZibraVDB Volume + 源文件 → 单独导入必缺东西 |
| 材质 | 各特效自带散材质，需 MI 化统一 |

## 3. 核心风险

1. **跨引擎迁移**：5.5公版→自研5.7.3，无5.5引擎则 Migrate 走不通；ZibraVDB 预编译 ABI 不可跨引擎复用。
2. **资产闭包断链**：残缺包缺 Volume/源文件，必须按依赖簇整体处理 + 补料。
3. **命名/重定向冲突**：散装并入需命名空间隔离 + Fix Up Redirectors（外包 Redirector 路径含拼写错误）。
4. **材质视觉一致性**：MI 替换要保参数映射准、效果不退化。
5. **规模化**：20+ 个，试点跑通后脚本化。

## 4. 流程（四阶段）

```
阶段0 摸底  →  阶段1 规范定义  →  阶段2 试点导入  →  阶段3 批量执行
（部分完成）   目录/命名/母材质    A45跑通沉淀SOP    放量+回归
```

## 5. 目录与命名规范

```
{主工程}/Content/FX/
  ├── _Master/      母材质 M_FX_Master、材质函数 MF_*、共享贴图
  ├── _Shared/      跨特效共享（LUT、噪声图）
  └── <EffectName>/
        ├── Level/ Geo/ Niagara/ PointCache/ Volume/ Material/ Texture/
```
前缀：`M_`母材质 / `MI_`实例 / `MF_`函数 / `NS_`Niagara / `T_`贴图 / `SM_`网格。

## 6. 导入方式（按形态分诊）

| 形态 | 方式 |
|---|---|
| 完整 uproject（A级） | 有同版本引擎则 Migrate；否则文件级整簇拷贝 + Fix Up |
| 散装 Content（B级残缺） | 整簇拷贝 + 补齐缺失 Volume + Fix Up Redirectors |
| ZibraVDB Volume | 用主工程 ZibraVDB 插件导入 `.zibravdb` 重建；无源则找外包补 |
| Houdini 源(.hip/HDA) | 主工程装 Houdini Engine 后重生成 |

> ⚠️ Migrate 受跨引擎限制，残缺包不可单独导入——核心是**按依赖闭包整簇处理**。

## 7. 材质统一策略

1. 先归纳散材质共性 → 建 `M_FX_Master`（分**普通粒子族** + **体积族** 两套，体积族用于 ZibraVDB）。
2. 每特效散材质替换为 `MI_<Effect>_xx`，只调参不改图。
3. 特例（自定义 HLSL）→ 母材质 Static Switch 分支或独立但纳入规范。
4. 替换后逐个视觉比对。

## 8. 里程碑

| 里程碑 | 产出 |
|---|---|
| M0 摸底 | 归档清单 + 版本依赖统计 + 散材质审查（部分完成） |
