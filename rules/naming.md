# Naming Rule — 文件/文件夹命名规范

> 适用于本工作区所有新建文件与文件夹。用户 2026-07-09 拍板。

## 规则

**文件名和文件夹名不出现中文**，一律用英文（含数字、`-`、`_`）命名。

- 适用范围：文件名、文件夹名本身
- 不适用范围：**文件内容**（正文、标题、注释照常用中文，不受影响）
- 目的：避免跨平台/跨工具（git、终端、脚本、链接引用）处理中文路径时的编码与转义问题

## 命名建议

- 优先用简洁的英文短语或缩写反映内容，如 `mcp-pitfalls.md`、`blockout-v2-measured.md`
- 多词用 `-`（kebab-case）或 `_`（snake_case）分隔，项目内保持风格一致即可，不强制统一到某一种
- 拿不准怎么翻译 → 可用拼音缩写兜底，但优先尝试英文

## 历史文件

- 本规则**不强制**批量重命名仓库里已存在的中文文件名（改动面大、会破坏历史链接/引用）
- 但如果顺手改到某个中文命名的旧文件（比如因为其他原因要编辑它），建议随手重命名并同步更新引用
- 新建文件/文件夹 **必须** 遵守本规则

## 维护

- 发现新建的中文文件名 → 按本规则重命名（用 `git mv` 保留历史）并同步所有引用处
