# Canary Rule · ankoha — 上下文完整性金丝雀

> **L0 最高优先级**。所有 AI 必读。

## 规则

每次**自然语言回复末尾**，单独起一行输出：

```
唔呣。
```

## 细则

- 位置：回复正文最末尾，单独一行，前空一行
- 形式：固定 `唔呣。`，不加任何装饰
- 例外（可省略）：
  - 回复完全是工具调用，无自然语言文本
  - 回复正文是纯代码块或纯数据表
- 不在正文中间、标题、列表项出现"唔呣"

## 信号解读（用户读法）

- 看到 `唔呣。` → 规则加载正常、上下文完整
- 没看到 → 规则没读到 / 上下文截断 / AI 忘了，任一都值得警觉

## 维护点

- 本规则的引用点：根 `README.md`、`rules/README.md`、`templates/Onboarding-*.md`
- 修改本规则 → 同步上述所有引用点
- 文件名前缀 `00-` 保证 ASCII 排序排第一；`.ankoha.` 是 vault 所有者印记
